azLeaflet
directive in module az.leaflet

Description

Instantiate a new leaflet map on the element, with options, which are not watched for changes

A class azavea-ng-leaflet-map is added to the root Leaflet map element

This directive does not handle addition of any basemaps, etc. This must all be done in child directives. See ./examples directory of the source for a detailed example.

Usage

as element:
<az-leaflet
       [width="{String}"]
       [height="{String}"]
       [options="{Object}"]>
</az-leaflet>

Directive info

  • This directive creates new scope.

Parameters

ParamTypeDetails
width
(optional)
String
  • Width of map in px or percentage
height
(optional)
String
  • Height of map in px or percentage
options
(optional)
Object
  • Default map options to instantiate map with. If not provided, az.leaflet.service:AZLeafletDefaults is used.

Events

  • invalidatesize

    Trigger 'az.leaflet.invalidatesize' in your application code to trigger a L.map.invalidateSize() call on the next $digest cycle
